El servicio $Localization permite localizar recursos de acuerdo a la configuración global permitiendo escalar el sistema a varios idiomas y recursos locales especificos por cada uno de ellos.

La carga de los recursos locales, se encuentra en la variable de configuración global application bajo la variable language y, es cargada en el paso de arranque de la aplicación angularJS.
Su ubicación se encuentra en /app/config/env/locales/{language_in_config}

Uso Basico

app.controller('MyController', function($scope, $Localization) { var locale = $Localization.get('labels.helloWorld'); console.log(locale); //Hola Mundo });

Uso Declarativo

Métodos

  • $Localization.get(name [,defaultValue]);

    Obtiene un recurso localizado y retorna su valor.

    Nombre Tipo Descripción
    name String

    Nombre del recurso a obtener

    defaultValue Object

    Valor por defecto a retornar si es que no se encuentra configurada el recurso objetivo.

    Retorno Descripción
    object

    Retorna el valor del recurso deseado; En el caso de no encontrarse la variable se ejecutará una excepción de control indicando que no existe tal variable.
    Si se encuentra establecido el parametro defaultValue y, no se encuentra la variable , la excepción será omitida y se retornará el valor de defaultValue

  • $Localization.exists(name);

    Verifica la existencia de un recurso localizado.

    Nombre Tipo Descripción
    name String

    Nombre del recurso a verificar.

    Retorno Descripción
    boolean

    Retorna verdadero si la variable se encuentra en cargada en el sistema; De lo contrario retornará falso.
